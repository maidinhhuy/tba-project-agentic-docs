## 2026-06-11 — Decisions từ Party Mode

### UI & Visual Direction
- **UI System:** shadcn/ui + TanStack Table (admin) + Recharts nếu cần chart
- **Layout:** Card-first cho customer portal; table cho admin dashboard
- **Visual tone:** Confident Clarity — purposeful whitespace, neutral deep/light + single accent color
- **Typography:** Inter hoặc Geist (TBD)

### Status States (Fixed — không để admin define)
8 states cố định theo state machine của Mary:
`SUBMITTED → ANALYZING → IN_DEVELOPMENT → AWAITING_REVIEW ⇄ IN_REVISION → FINALIZING → DELIVERED`
+ `CANCELLED` từ bất kỳ state nào (trừ DELIVERED)
- **2-tier architecture:** Customer thấy fixed states; Admin có internal progress notes (flexible)
- Color semantics: xanh dương nhạt (submitted) → cam (reviewing) → xanh lá active (in progress) → vàng (needs input) → xanh lá đậm (completed) → đỏ nhạt (cancelled)

### Customer Experience
- **Milestone:** Read-only (customer chỉ xem, không approve/reject)  ← *superseded by Epic 8 update below*
- **Revision counter:** Hiển thị công khai trên project detail

### Information Architecture (Customer)
3 màn hình: Dashboard (project list) → Project Detail → Submit Form

## 2026-06-20 — Epic 8 UX Update

### Customer Project Detail
- **Milestone display:** Horizontal stepper (Option A) — full-width, circle indicators (48px default / 56px active), connector line, active step expands a panel showing deliverable link + note + Approve/Revision actions. *Supersedes "milestone read-only" from 2026-06-11.*
- **Description:** Collapsed by default (3 lines via line-clamp), "Xem thêm" / "Thu gọn" toggle, arrow icon animates on open/close
- **Reference field:** External link with icon below description, always visible, hidden when null/empty
- **Edit button:** In project header → `/projects/:id/edit`
- **Revision counter:** Moved to header alongside status badge (not below stepper)

### Admin Project Management
- **Tab layout confirmed:** Tab "Thông tin" (description + reference) · Tab "Quản lý" (milestone rows, status form, review history)
- **Admin Tab 1 description:** Same collapsible pattern as customer side
- **Delivery button visibility:** Hidden (not disabled) unless all 3 conditions met: project=IN_DEVELOPMENT AND milestone=ACTIVE AND review_status=PENDING_DELIVERY
- **Milestone row badges:** PENDING_REVIEW→yellow, APPROVED→green, REVISION_REQUESTED→orange
- **Deliver form:** Modal (shadcn Dialog), not sheet/inline. URL + file in one deliverable group with "Hoặc" divider; note field separate below. Footer has informational note "Khách hàng sẽ nhận email thông báo ngay."

### State machine additions (Epic 8)
- `IN_DEVELOPMENT → AWAITING_REVIEW`: auto-triggered by system when Admin delivers milestone
- `AWAITING_REVIEW → IN_DEVELOPMENT`: Customer approves non-final milestone
- `AWAITING_REVIEW → DELIVERED`: Customer approves final milestone
- `AWAITING_REVIEW → IN_REVISION`: Customer requests revision (decrements revision_count)
